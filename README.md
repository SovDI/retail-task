Использование ИИ (Gemini)
Проект реализован при поддержке Gemini. Модель использовалась для:

Написания Python-скриптов синхронизации (RetailCRM API + Supabase SDK).

Разработки дашборда на HTML/Chart.js.

Помощи в деплое на Vercel.

Основные промпты:
«Напиши скрипт для выгрузки заказов из RetailCRM в Supabase на Python».

«Сделай уведомление в Telegram для заказов дороже 50 000 ₸».

«Создай HTML-страницу с графиком Chart.js, подтягивающим данные из Supabase».

Проблемы и решения:
Что было: В названии проекта была лишняя буква «u», а в конце адреса стояло привычное .com вместо нужного для этой базы .co.
Как решили: Пришлось несколько раз буквально под микроскопом сравнивать адрес в коде и в настройках, пока не нашли лишний символ.

Что было: Использовался короткий публичный ключ, который подходит для одних задач, но не работает для отображения графиков на сайте.
Как решили: Нашли в настройках специальный длинный ключ (Legacy anon key). Это своего рода «полный пропуск», после вставки которого данные мгновенно начали отображаться.

Главная трудность была в том, чтобы понять, где именно произошел сбой: в коде Python, в настройках GitHub или в самом браузере.

Как решили: Использовали консоль разработчика как детективный инструмент. Каждая красная строчка в консоли давала подсказку, что именно исправлять следующим шагом — сначала адрес, потом ключ, потом кэш.
